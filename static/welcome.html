This is a ToolFactory Development Galaxy server


Please do not expose on the public internet. None of the normal public security layers have been added. This is a persistent but throwaway development Galaxy instance using the sqlite default clone.

The ToolFactory uses ordinary Galaxy forms to generate real Galaxy tools.
Complicated tools need a skilled developer using the recommended software. Simple tools can be very useful when you need them, and most simple script based tools can be created by capable developers who have not yet acquired the specialised skills, using this special Galaxy server. It generates new tools, from existing scripts of any kind, by filling in a Galaxy form.
Note: the ToolFactory is a terrible way to get a script working properly!. Get it working in a shell, with small test data samples on your local machine first. Those working test files are needed in your history. They become example inputs when you are completing the new tool ToolFactory form. They also become tool test data in the generated Toolshed ready archive file.
Dysfunctional, broken scripts make dysfunctional broken tools so only start a new tool when the new script works correctly on your command line with small test data

The preinstalled examples show some of the ways that different tool patterns can be created.
The ToolFactory can turn any working script into an ordinary, shareable Galaxy tool and Toolshed ready archive. Examples include (trivial) scripts for diverse interpreters including Lisp, Prolog, Bash, Python and R. Any conda dependency can be run and a command line constructed, or entered as an over-ride string, as in the BWA examples
For any generated tool in the history, the ToolFactory form can be recreated as it was. The form will be set up, ready for you to change things to create tools you could use. Change the tool id to create a new tool from one of the existing ones. Generated tools appear on the local tool menu - refresh your browser to see them after the ToolFactory job finishes.
There is an installed workflow that will install all the ToolFactory examples
Assign the input files from the initial history to correctly match the form data name labels or it won't work!
Galaxy's UI is not ideal for building tools with dozens of parameters and dozens of files. Much better to build complicated tools in the usual manual way, if you have those skills.
Until you do the training, the ToolFactory may be handy for creating useful Galaxy tools from your own working scripts.
Scripts can use any Conda dependencies, such as BWA/samtools examples, and the Lisp, Prolog and bash interpreter examples.

Ideally with just a handful of i/o files and parameters. Building complicated tools requires a skilled developer and the conventional infrastructure.
